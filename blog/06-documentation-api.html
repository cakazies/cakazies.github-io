Membuat Documentasi API di Repo ? Mudah kok

30 Januari 2020

Oke Guyss, sudah 3 minggu belum membuat artikel lagi heheh karena dikantor masih sibuk dengan kerjaan ya biasa wkwkwkw sok sibuk anjeeerr, oke kali ini mungkin saya akan share untuk bagaimana membuat dokumentasi API khususnya untuk backend yah. akan tetapi bisa juga kok buat bahan bacaan untuk front end atau yang yang lain mungkin bisa jadi untuk referensi

Fungsi Dokumentasi

okey, apa sih fungsinya dokumentasi ? mungkin ini detailnya salah satu point yang saya telah jabarkan di artikel saya sebelumnya clean code, mungkin di situ di singgung dengan salah satu clean code adalah adanya documentasi, lalu fungsinya dokumentasi itu sendiri apa sih, oke fungsinya dokumentasi itu dari kata dokumentasi dari sumber wikipedia adalah tata cara untuk menggunakan sesuatu, dalam hal ini yang kita bicarakan adalah repo dari sebuah code jadi bisa digabungkan pengertiannya adalah tata cara untuk menggunakan sebuah library baik itu API ataupun apa saja fitur dokumentasi itu sendiri, dalam hal ini kita akan membahas bagaimana membuat dokumentasi yang mudah untuk dibaca.

Prolog

Sebelum membuat dokumentasi kita harus tau apa saja yang ada di dalam repo yang akan kita buat itu, minimal cara menggunakan repo itu sendiri, okey mungkin contoh-contoh yang saya akan buat ini akan saya contohkan di github akan tetapi bisa digunakan untuk yang lain juga. okey biasanya dalam sebuah company atau startup mungkin beberapa menyebutnya engineer spec, nah itu sama dengan documentation API itu sendiri, hanya saja biasanya menggunakan Jira software atau yang biasa menggunakan PDF tergantung dari kebutuhan company itu sendiri, akan tetapi juga biasanya digunakan fitur dari github/ gitlab yaitu wiki balik lagi tergantung perusahaannya untuk kebutuhannya bagaimana.

Documentation

Oke langsung saya yah, pertama bisa tentukan apa saja yang akan menjadi API yang sudah kamu selesaikan contohnya ada beberapa nih

  • API Get Room
  • API Create Room
  • API Update Room
  • API Delete Room
  • API Get Detail Room

contohnya kamu punya 5 API yang akan digunakan, seperti yang diatas, jika kamu sering menggunakan POSTMAN atau Insomnia sudah biasa mengenal istilah authorization, params, data body, URL, host dan lain-lain

Step pertama

Jelaskan setiap endpoint/API yang kamu buat di setiap penjelasan API, contohnya jika kamu membuat API Get Room, API itu digunakan untuk apa saja, dan digunakannya untuk apa, Methodnya apa, Urlnya apa, bisa digunakan tabel untuk memperindah dan mempermudah penjelasan, contohnya disini

Command Method Routes
Get Room GET api/v1/rooms

Step Kedua

Dalam setiap API kebanyakan menggunakan Authorization, kecuali API yang consumenya untuk public, untuk step kedua adalah jelaskan setiap API memakai authorization apa, apakah menggunakan token, Basic Auth, Bearer atau yang lainnya.

Step three

Jika ada parameter yang digunakan tuliskan juga, bisa dijelaskan dalam bentuk table atau mungkin bisa list saja,contohnya adalah disini

KEY VALUE
page 1,10,9,all
filter 17,72,100
shortby id,desc
limit 10,100

Step Four

Jelaskan juga apakah pada API yang akan kamu buat itu menggunakan data Request,data yang bisa di kirim bisa bermacam-macam bisa berupa form, graphql, json akan tetapi kebanyakan data request menggunakan JSON, berikut contohnya

Step Five

Setiap request yang akan dilakukan pastinya akan mendapatkan response dari API itu sendiri, entah success atau gagal atau yang lain, tergantung dari request kita sendiri apakah ada something yang kurang atau bagaimana:

  • Request Success
  • Request ID not found
  • Request Authorization Required
  • Request Server Error
